## 📝 Object-Oriented Programming (OOP) Notes

> This document contains quick reference notes and important Object-Oriented Programming concepts learned in C++.

##

Object-Oriented Programming is a programming paradigm that models real-world entities using **objects** and **classes**.

Its main objective is to improve code **reusability**, **modularity**, **security**, and **maintainability**.

---

## 📌 Classes

A **class** is a blueprint or template used to create objects.

It defines:

- Data members (Properties)
- Member functions (Methods)

```cpp
class Student{
public:
    string name;
    int age;

    void display();
};
```

---

## 📌 Objects

An **object** is an instance of a class.

Each object has:

- Properties (Variables)
- Behaviors (Functions)

```cpp
Student s1;
```

---

## 📌 Access Modifiers

Access modifiers determine the accessibility of data members and member functions.

| Modifier | Accessibility |
|-----------|---------------|
| `private` | Accessible only inside the class *(Default in C++)* |
| `public` | Accessible from anywhere |
| `protected` | Accessible inside the class and derived classes |

---

## 📌 Encapsulation

Encapsulation is the process of combining **data** and **methods** into a single unit (class) while restricting direct access to sensitive data.

Benefits:

- Data hiding
- Better security
- Controlled access

Usually implemented using:

- Private variables
- Public Getter functions
- Public Setter functions

---

## 📌 Getters & Setters

Getter functions return private data.

Setter functions modify private data.

```cpp
void setSalary(double s){
    salary = s;
}

double getSalary(){
    return salary;
}
```

---

## 📌 Constructors

A constructor is a special member function that is automatically called whenever an object is created.

Characteristics:

- Same name as the class
- No return type
- Executes automatically

```cpp
Student s1;
```

---

**Non-Parameterized Constructor**

Initializes objects with default values.

```cpp
Student(){
    name = "Unknown";
}
```

---

**Parameterized Constructor**

Accepts values during object creation.

```cpp
Student(string n, int a){
    name = n;
    age = a;
}
```

---

**Constructor Overloading**

A class can have multiple constructors with different parameter lists.

Example of **Compile-Time Polymorphism**.

```cpp
Student();

Student(string name);

Student(string name, int age);
```

---

## 📌 The `this` Pointer

`this` is a special pointer that refers to the current object.

Useful when parameter names and member variable names are identical.

```cpp
this->name = name;
this->age = age;
```

Equivalent to

```cpp
(*this).name = name;
```

---

## 📌 Copy Constructor

Creates a new object using an existing object.

**Default Copy Constructor**

Automatically generated by the compiler.

```cpp
Student s2(s1);
```

##

**Custom Copy Constructor**

Defined by the programmer.

```cpp
Student(Student &obj){
    name = obj.name;
}
```

Used when copying dynamic memory safely.

---

## 📌 Shallow Copy

Copies only the memory addresses.

Both objects point to the same memory location.

```
Object A  --------\
                   > Same Memory
Object B  --------/
```

Problems:

- Shared data
- Unexpected modifications
- Double deletion

---

## 📌 Deep Copy

Creates completely independent copies.

Each object has its own memory.

```
Object A -----> Memory A

Object B -----> Memory B
```

Benefits:

- Safe
- Independent objects
- Prevents memory issues

---

## 📌 Dynamic Memory Allocation

Memory allocated during runtime using:

```cpp
new
```

Released using:

```cpp
delete
```

Example

```cpp
int* ptr = new int;
```

---

## 📌 Four Pillars of OOP

| Concept | Description |
|----------|-------------|
| Encapsulation | Wrapping data and methods together |
| Abstraction | Hiding implementation details |
| Inheritance | Acquiring properties from another class |
| Polymorphism | One interface, multiple implementations |

---

## 📌 Compile-Time vs Run-Time Polymorphism

| Compile-Time | Run-Time |
|--------------|----------|
| Function Overloading | Function Overriding |
| Constructor Overloading | Virtual Functions |
| Faster | More Flexible |

---

## 📌 Advantages of OOP

- Code Reusability
- Better Security
- Easy Maintenance
- Real-world Modeling
- Modularity
- Scalability

---

## 📌 Common Keywords

| Keyword | Purpose |
|----------|---------|
| `class` | Create a class |
| `public` | Public access |
| `private` | Private access |
| `protected` | Protected access |
| `this` | Current object pointer |
| `new` | Dynamic memory allocation |
| `delete` | Free allocated memory |

---

## 💡 Key Takeaways

- Classes act as blueprints; objects are their instances.
- Constructors initialize objects automatically.
- Encapsulation protects data using access modifiers.
- Getter and Setter functions provide controlled access.
- The `this` pointer refers to the current object.
- Copy constructors duplicate objects safely.
- Deep Copy avoids shared memory problems.
- OOP improves code organization, reusability, and scalability.
